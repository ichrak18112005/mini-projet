Projet : Application Client / Serveur DevOps

Cette application permet de gérer des tâches avec une architecture Client/Serveur TCP.
Elle supporte plusieurs clients et peut être exécutée avec ou sans Docker.

Contenu du projet :
- client_v1 : client simple
- client_v2 : client amélioré
- client3 : client pour Docker (2 clients configurés)
- client3SD : client pour exécution sans Docker, multi-clients possibles
- serveur_v1 : serveur simple
- serveur_v2 : serveur amélioré avec sauvegarde JSON
- serveur3 : serveur final avec Docker, multi-clients et persistance JSON

Fonctionnalités :
- Ajouter une tâche
- Lister les tâches
- Supprimer une tâche par ID
- Modifier le statut d’une tâche (TODO -> DOING -> DONE)
- Multi-clients simultanés

Lancement sans Docker :
1. Lancer le serveur :
   python serveur3/serveur3.py
2. Lancer un ou plusieurs clients :
   python client3SD/client3SD.py

Lancement avec Docker :
1. Construire et lancer les conteneurs :
   docker compose up --build
2. Lancer les clients définis (2 clients) :
   docker compose run client1
   docker compose run client2
